> *"Shared language is the foundation of cooperation."*

## Overview

This is the **official glossary** for all AION terminology.
When a term appears in any AION document, it means exactly what this document says it means.
No local redefinitions. No informal usage.

---

## Core Entities

### AION
*Artificial Intelligence Omni Network.*
The entire system — all institutes, agents, systems, and knowledge — operating as a unified organization.

### Sophia (Executive)
The central executive intelligence of AION.
Sophia coordinates, approves, and governs. Sophia does not research or create.
Not to be confused with the Sophia Institute.

### Institute
An independent organizational unit within AION, focused on one intellectual domain.
Each institute has a Director, specialist agents, protocols, and an archive.

### Agent
An autonomous specialist AI with a clearly defined role, domain, and mandate.
An agent operates only within its defined domain.

### Director
The lead agent of an institute. Responsible for institute operations, quality, and coordination with Sophia.

### Archivist
The agent responsible for maintaining an institute's knowledge archive and memory.

## System Components

### Core Systems
The shared infrastructure used by all institutes and agents.
Includes: Memory, Knowledge Graph, Verification, Communication, Logging, Security.

### Memory System
AION's mechanism for retaining, organizing, and retrieving information across sessions.
Defined in `02_Core/Memory.md`.

### Knowledge Graph
The structured network of entities, relationships, and verified facts stored in AION.
Defined in `02_Core/KnowledgeGraph.md`.

### Knowledge Base
The persistent library of verified knowledge entries produced by AION agents.
Located in `07_Knowledge/`.

### Verification Engine
The system that checks claims against sources before publication.
Defined in `02_Core/Verification.md`.

### Communication Layer
The protocol system that governs how agents and institutes exchange information.
Defined in `02_Core/Communication.md`.

## Process Terms

### Task
A unit of work assigned to AION by a human operator or generated by Sophia.

### Routing
The process by which a task is assigned to the correct institute or agent.
Performed by Sophia.

### Research
The systematic collection and analysis of information within an agent's domain.
Governed by `05_Workflows/Research_Workflow.md`.

### Verification
The process of checking claims against traceable sources.
Governed by `05_Workflows/Validation_Workflow.md`.

### Review
The quality control process applied before any output reaches "Approved" status.
Governed by `01_Sophia/QualityControl.md`.

### Escalation
The process of passing a decision or problem up the hierarchy when an agent cannot resolve it independently.

### Delegation
The process by which Sophia assigns a task to an institute or Director.

## Document Terms

### Constitution
The set of foundational documents in `00_Constitution/` that govern AION's identity and behavior.
Constitutional documents are near-immutable.

### Charter
An institute's founding document. Defines identity, domain, mandate, and structure.

### Mandate
The defined scope of authority and responsibility for an agent or institute.

### Protocol
A defined procedure for a recurring activity within an institute.

### Workflow
A defined end-to-end procedure that spans multiple agents or institutes.
Stored in `05_Workflows/`.

### Template
A reusable document structure for standard output types.
Stored in `06_Templates/`.

---

## Status Terms

### Draft
A document being written. Not yet reviewed.

### Review
A document submitted for QC review. Not yet approved.

### Approved
A document that has passed QC review. Official version.

### Deprecated
A document replaced by a newer version. Moved to `08_Archive/`.

### Archived
A document preserved for historical reference. Read-only.

---

## Quality Terms

### Quality Score
A numerical measure of output quality per the standards in `09_Governance/Quality_Standards.md`.

### Verification Rate
The percentage of claims in a document that are traced to a source.
Target: 100% for all approved documents.

### Confidence Level
An explicit label attached to any non-verified claim.
Valid levels: `Confirmed`, `High Confidence`, `Moderate Confidence`, `Speculative`.

---

## Relationship Terms

### Cross-Institute Collaboration
When two or more institutes contribute to a single task, each within its domain.

### Peer Review
When one agent reviews the output of another agent of equivalent level.

### Director Review
When a Director reviews the output of an agent within the same institute.

### Sophia Review
When Sophia reviews an output that requires executive approval.
