# ROADMAP
## AION Development Plan

> "Build slowly. Build correctly. Build to last."

---

## Philosophy

AION is not a sprint — it is a civilization.
Each version adds permanent, verified, quality-controlled components.
No version releases until it meets the quality standards defined in `09_Governance/Quality_Standards.md`.

## Design Constraints

These constraints apply to **all versions**:

1. No version ships with unverified content
2. Every agent has one clearly defined domain
3. No agent makes decisions outside its mandate
4. All cross-institute work goes through Sophia
5. Quality is measured before release
